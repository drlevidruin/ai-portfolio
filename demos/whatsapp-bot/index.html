Text the school. Claude confirms. Sheets get written.

Pardes staff (about 25 teachers, rabbis, and admin) text the school WhatsApp number. V1 handles two intents: time-off requests (piped into the existing SIS) and supply requests (logged to a sheet plus emailed to the office). Claude extracts the details, sends back a confirmation card, and nothing gets written until staff tap "Yes, submit." This is a simulation of that exact flow.

What the system is doing

Every message goes through this exact 4-stage flow. Meta requires a 200 OK within 10 seconds, so a Cloudflare Worker sits in front and Apps Script processes asynchronously on a 1-minute trigger.

Real-world details: Model is Claude Haiku 4.5 at roughly $0.0017 per message based on observed usage. The Cloudflare Worker verifies Meta's HMAC-SHA256 signature and returns 200 to Meta via event.waitUntil(). Apps Script checks a proxy_token before doing any work. Messages are dedup'd by wa_message_id in the Message_Queue sheet. Confirmation cards include a nonce so double-taps cannot double-submit.
